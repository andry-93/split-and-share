# Split & Share — UI Tone Guide (EN)

## Goal
Use clear, action-oriented microcopy that helps users finish tasks fast.

## Core rules
- Prefer active voice.
- Use short labels (1–3 words when possible).
- Start with verbs for CTAs and recovery messages.
- Avoid passive phrasing like `is owed`, `is due`, `unable to`.
- Keep debt language consistent across screens and PDF.

## Approved debt vocabulary
- `Collect` — money the user should receive.
- `Pay` — money the user should pay.
- `Mark paid` — action for completed transfer.
- `Settled` — no open balance.
- `Balances` — preferred umbrella term over `Debts` in user-facing copy.

## Replace map (legacy -> approved)
- `You get` -> `Collect`
- `You owe` -> `Pay`
- `You are owed` -> `Collect`
- `Mark as paid` -> `Mark paid`
- `All settled` -> `Settled`
- `No debts yet` -> `No balances yet`
- `Add expenses to see who owes whom.` -> `Add expenses to see balances.`
- `owes` / `pays` (row subtitle) -> `pay`

## Empty states
- Expenses: `Add your first expense`
- People (global): `Add your first person`
- People (event): `Add people to this event`
- Debts: `No balances yet`

## Error/recovery style
Pattern:
- Primary action + `again`
- Optional quick fallback instruction

Examples:
- `Generate the PDF report again.`
- `Share the PDF again.`
- `Save the PDF again.`
- `Load contacts again.`
- `Try again. Restart the app if this continues.`

## CTA style
Preferred:
- `Save`, `Create`, `Update`, `Delete`, `Share`, `Collect`, `Pay`, `Mark paid`

Avoid:
- `Unable to ...`
- `Failed to ...`
- Long explanatory button labels

## Scope note
This guide defines **English** copy style only. Localization can map these terms later, but source EN strings should follow this guide.
